# Rubric: a11y

Judge accessibility from the screenshot. (Programmatic axe-core checks happen separately in the design-system-agent; this rubric judges what an axe scan can't see — actual readability and interaction affordances.)

## Dimensions

- **contrast_sufficient** (1–10): visible text passes WCAG AA contrast against its background. Frosted-glass overlays must darken enough to support legible text. 10 = all text legible; 1 = light-on-light or dark-on-dark visible.
- **focus_visible** (1–10): if the screenshot shows a focused element, its focus indicator is visible. If no element is focused (initial load), score 10.
- **target_size_sufficient** (1–10): clickable elements (buttons, drawer entries, nav links) appear large enough to tap on a mobile viewport (≥44×44 px equivalent). Score relative to viewport.
- **landmark_structure** (1–10): page appears to have semantic structure — distinct header / main / footer regions, headings used for hierarchy not styling.

## Output schema

```json
{
  "contrast_sufficient": <1-10>,
  "focus_visible": <1-10>,
  "target_size_sufficient": <1-10>,
  "landmark_structure": <1-10>,
  "reasoning": "<2-4 sentences>",
  "lowest_dimension": "<key>",
  "would_generate_candidate": <true if any < 7 else false>
}
```
